# Sill v1.1 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 설정 창(자동 실행·스택 위치), 주 모니터 고정, 핫키 실패 알림.

**Architecture:** `AppSettings` ObservableObject 싱글턴이 UserDefaults/SMAppService를 감싸고, 설정 창(SwiftUI)과 StackPanelController가 이를 구독한다.

**Tech Stack:** Swift, SwiftUI, AppKit, ServiceManagement. 의존성 추가 없음.

**Spec:** `docs/superpowers/specs/2026-07-05-v1.1-design.md`

## Global Constraints

- macOS 13+, 외부 패키지 의존성 금지, 자동 단위 테스트 없음 — 게이트는 `swift build` 성공 (수동 GUI 검증은 컨트롤러가 마지막에 일괄).
- 기존 파일의 다른 동작(캡처, DnD, Sparkle 등)은 건드리지 않는다.

**Files:**

**Interfaces:**
- Produces: `AppSettings.shared` — `@Published var stackSide: AppSettings.StackSide` (`.left`/`.right`, UserDefaults 키 "stackSide"), `@Published var launchAtLogin: Bool`. Task 2가 `$stackSide`를 구독한다.
- Produces: `SettingsWindowController` — `func show()`.

- [ ] **Step 1: AppSettings.swift 작성**

```swift
import AppKit
import Combine
import ServiceManagement

final class AppSettings: ObservableObject {
    static let shared = AppSettings()

    enum StackSide: String {
        case left, right
    }

    @Published var stackSide: StackSide {
        didSet { UserDefaults.standard.set(stackSide.rawValue, forKey: "stackSide") }
    }

    /// SMAppService 상태가 진실의 원천 — 등록/해제 실패 시 토글 원복
    @Published var launchAtLogin: Bool {
        didSet {
            guard oldValue != launchAtLogin else { return }
            do {
                if launchAtLogin {
                    try SMAppService.mainApp.register()
                } else {
                    try SMAppService.mainApp.unregister()
                }
            } catch {
                NSLog("로그인 시 자동 실행 변경 실패: \(error.localizedDescription)")
                launchAtLogin = oldValue
            }
        }
    }

    private init() {
        stackSide = StackSide(rawValue: UserDefaults.standard.string(forKey: "stackSide") ?? "") ?? .right
        launchAtLogin = SMAppService.mainApp.status == .enabled
    }
}
```

- [ ] **Step 3: SettingsWindowController.swift 작성**

```swift
import AppKit
import SwiftUI

final class SettingsWindowController {
    private var window: NSWindow?

    func show() {
        if window == nil {
            let win = NSWindow(
                contentRect: .zero,
                styleMask: [.titled, .closable],
                backing: .buffered,
                defer: false
            )
            win.title = "Sill 설정"
            win.contentView = NSHostingView(rootView: SettingsView(settings: .shared))
            win.isReleasedWhenClosed = false
            win.center()
            window = win
        }
        NSApp.activate(ignoringOtherApps: true) // accessory 앱이라 명시적으로 앞으로
        window?.makeKeyAndOrderFront(nil)
    }
}
```

### Task 2: 주 모니터 고정 + 스택 위치 반영

**Files:**
- Modify: `Sources/Sill/StackPanelController.swift`

**Interfaces:**
- Consumes: `AppSettings.shared.$stackSide` (Task 1).

- [ ] **Step 1: StackPanelController 수정**

프로퍼티 `private var cancellable: AnyCancellable?`를 다음으로 교체:

```swift
    private var cancellables: Set<AnyCancellable> = []
    private var count = 0
```

`init`의 구독부(`cancellable = store.$screenshots...`)를 다음으로 교체:

```swift
        store.$screenshots
            .receive(on: DispatchQueue.main)
            .sink { [weak self] shots in
                self?.count = shots.count
                self?.layout()
            }
            .store(in: &cancellables)

        AppSettings.shared.$stackSide
            .dropFirst()
            .receive(on: DispatchQueue.main)
            .sink { [weak self] _ in self?.layout() }
            .store(in: &cancellables)
```

`layout(count:)` 메서드를 다음으로 교체 (주 모니터 고정 + 좌/우):

```swift
    private func layout() {
        guard count > 0, let screen = NSScreen.screens.first else {
            panel.orderOut(nil)
            return
        }
        let visible = screen.visibleFrame
        let margin: CGFloat = 16
        let height = min(CGFloat(count) * itemHeight + 16, visible.height - margin * 2)
        let x = AppSettings.shared.stackSide == .right
            ? visible.maxX - panelWidth - margin
            : visible.minX + margin
        // 하단 앵커: y는 하단 마진에 고정, 항목이 늘면 위로 자람
        panel.setFrame(
            NSRect(x: x, y: visible.minY + margin, width: panelWidth, height: height),
            display: true
        )
        panel.orderFrontRegardless()
    }
```

주의: `AppSettings.$stackSide` 구독의 `.dropFirst()`는 초기값으로 인한 불필요한 첫 layout 호출을 막는다. `NSScreen.screens.first`는 메뉴바가 있는 주 디스플레이다.

- [ ] **Step 1: HotkeyManager.register가 OSStatus 반환**

기존 `register` 메서드를 다음으로 교체:

```swift
    @discardableResult
    func register(keyCode: UInt32, modifiers: UInt32, id: UInt32, handler: @escaping () -> Void) -> OSStatus {
        handlers[id] = handler
        var ref: EventHotKeyRef?
        let hotKeyID = EventHotKeyID(signature: OSType(0x5353_5441), id: id) // "SSTA"
        let status = RegisterEventHotKey(keyCode, modifiers, hotKeyID, GetApplicationEventTarget(), 0, &ref)
        hotKeyRefs.append(ref)
        return status
    }
```

        let interactiveStatus = hotkeys.register(
            keyCode: UInt32(kVK_ANSI_4),

        }
