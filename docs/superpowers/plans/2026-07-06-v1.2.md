# Sill v1.2 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 저장 위치·형식 설정, 썸네일 우클릭 메뉴(복사/저장), 타이머 전체 캡처.

**Architecture:** AppSettings에 imageFormat/saveDirectory 추가 → CaptureService가 반영. Screenshot에 isManaged 플래그로 삭제 정책 분기. 우클릭은 기존 DragNSView에 NSMenu.

**Tech Stack:** Swift, SwiftUI, AppKit. 의존성 추가 없음.

**Spec:** `docs/superpowers/specs/2026-07-06-v1.2-design.md`

## Global Constraints

- macOS 13+, 외부 패키지 의존성 금지, 자동 단위 테스트 없음 — 게이트는 `swift build` 성공 (수동 GUI 검증은 컨트롤러가 일괄).
- 기존 동작 불변: 임시 폴더(기본값) 캡처의 ✕/드래그 시 파일 삭제 의미론, DnD, Sparkle, 핫키.
- 사용자 지정 폴더의 파일은 어떤 경로로도 앱이 삭제하지 않는다.

`Screenshot` 구조체에 필드 추가:

```swift

        let id = UUID()
        let url: URL
        let image: NSImage

```

`add(url:)`에서 insert 줄을 다음으로 교체:

```swift

        screenshots.insert(Screenshot(url: url, image: image, isManaged: isManaged), at: 0)
```

`remove(_:)`를 다음으로 교체:

```swift
    func remove(_ shot: Screenshot) {
        if shot.isManaged {
            try? FileManager.default.removeItem(at: shot.url)
        }
        screenshots.removeAll { $0.id == shot.id }
    }
```

`removeAfterDrag(_:)`의 지연 삭제 블록을 다음으로 교체:

```swift
        guard shot.isManaged else { return }
        // ponytail: 60초 유예 후 삭제 — 느린 수신 앱(업로드 등)이 문제 되면 보존으로 전환
        DispatchQueue.main.asyncAfter(deadline: .now() + 60) {
            try? FileManager.default.removeItem(at: shot.url)
        }
```

파일 끝(클래스 안)에 메서드 2개 추가:

```swift
    func copy(_ shot: Screenshot) {
        let pasteboard = NSPasteboard.general
        pasteboard.clearContents()
        pasteboard.writeObjects([shot.url as NSURL, shot.image])
    }

    func saveAs(_ shot: Screenshot) {
        let panel = NSSavePanel()
        panel.nameFieldStringValue = shot.url.lastPathComponent
        NSApp.activate(ignoringOtherApps: true) // accessory 앱이라 패널을 앞으로
        panel.begin { response in
            guard response == .OK, let destination = panel.url else { return }
            try? FileManager.default.removeItem(at: destination) // 패널이 덮어쓰기 확인을 이미 받음
            try? FileManager.default.copyItem(at: shot.url, to: destination)
        }
    }
```

```swift
    private func chooseSaveDirectory() {
        let panel = NSOpenPanel()
        panel.canChooseDirectories = true
        panel.canChooseFiles = false
        panel.canCreateDirectories = true
        panel.prompt = "선택"
        NSApp.activate(ignoringOtherApps: true)
        panel.begin { response in
            guard response == .OK, let url = panel.url else { return }
            settings.saveDirectory = url
        }
    }
```

- [ ] **Step 2: DragArea/DragNSView에 우클릭 메뉴**

`DragArea`에 프로퍼티 추가 및 updateNSView 갱신:

```swift
struct DragArea: NSViewRepresentable {
    let shot: ScreenshotStore.Screenshot
    let dragEnded: () -> Void
    let copy: () -> Void
    let saveAs: () -> Void

    func makeNSView(context: Context) -> DragNSView { DragNSView() }

    func updateNSView(_ view: DragNSView, context: Context) {
        view.url = shot.url
        view.image = shot.image
        view.onDragEnded = dragEnded
        view.onCopy = copy
        view.onSaveAs = saveAs
    }
}
```

`DragNSView`에 프로퍼티와 우클릭 처리 추가 (`onDragEnded` 아래):

```swift
    var onCopy: (() -> Void)?
    var onSaveAs: (() -> Void)?
```

`mouseUp` 아래에 추가:

```swift
    override func rightMouseDown(with event: NSEvent) {
        let menu = NSMenu()
        let copyItem = NSMenuItem(title: "클립보드에 복사", action: #selector(copyAction), keyEquivalent: "")
        copyItem.target = self
        menu.addItem(copyItem)
        let saveItem = NSMenuItem(title: "다른 이름으로 저장...", action: #selector(saveAsAction), keyEquivalent: "")
        saveItem.target = self
        menu.addItem(saveItem)
        NSMenu.popUpContextMenu(menu, with: event, for: self)
    }

    @objc private func copyAction() {
        onCopy?()
    }

    @objc private func saveAsAction() {
        onSaveAs?()
    }
```
