# Sill v1.8 — DeepL 번역 엔진 지원 설계

날짜: 2026-07-06
상태: 승인됨

## 목표

번역 엔진을 애플 온디바이스(기본)/DeepL API 중 선택. DeepL은 API 키를 키체인에 저장하고
free/pro 엔드포인트를 키 접미사로 자동 판별. 실패/키 없음은 카드에 오류 표시.

## 범위

포함:
- `DeepLTranslator` — 기존 `Translator` 프로토콜 구현
- 설정에 엔진 선택 + DeepL API 키 입력(키체인 저장)
- 방향 규칙(한국어→영어, 그 외→한국어)은 두 엔진이 공유

제외:
- DeepL formality/glossary 등 고급 옵션 (YAGNI)
- 엔진별 대상 언어 커스터마이즈 (자동 규칙 유지)
- DeepL 실패 시 애플로 자동 폴백 (오류를 명확히 보여주는 쪽 선택)

## 제약

- 외부 패키지 의존성 금지 — Security(키체인)·Foundation(URLSession)만 사용.
- 자동 테스트 없음 — 게이트는 `swift build` + 수동 검증. 단, DeepL 방향/엔드포인트 판별은 순수 로직이라 self-check 하나 남긴다.
- 기존 애플 번역 흐름(`T` 키, `.translationTask`) 불변.

## 컴포넌트

### Keychain (신규, ~30줄)
- Security 프레임워크 generic password get/set/delete. service=번들 ID, account=`"deepl-api-key"`.
- `static func string(for account:) -> String?`, `static func set(_ value: String?, for account:)`.

### Translator.swift (변경)
- `enum TranslationLanguage { static func isKorean(_ text: String) -> Bool }` — `NLLanguageRecognizer` dominant 감지, 두 엔진 공유.
- `AppleTranslator`는 이 헬퍼 사용하도록 정리.
- `DeepLTranslator: Translator` (struct):
  - `init(apiKey: String)`.
  - `translate(_:)`: 키 비었으면 `TranslationError.missingDeepLKey` throw. 감지→target(`EN-US`/`KO`).
    엔드포인트: 키가 `:fx`로 끝나면 `api-free.deepl.com`, 아니면 `api.deepl.com`.
    `POST /v2/translate`, 헤더 `Authorization: DeepL-Auth-Key <키>`,
    form body `text`+`target_lang`. HTTP 비정상이면 `.deepLHTTP(status)` throw.
    응답 JSON `translations[0].text` 반환.
- `enum TranslationError: LocalizedError` — `missingDeepLKey`, `deepLHTTP(Int)`, `emptyResponse`. 한국어 메시지.

### AppSettings (변경)
- `enum TranslationEngine: String { case apple, deepl }`.
- `@Published var translationEngine`(UserDefaults).
- `@Published var deepLAPIKey: String` — `didSet`에서 Keychain 저장, `init`에서 Keychain 로드.

### GeneralSettingsView (변경)
- "번역 엔진" Picker(애플/DeepL). DeepL 선택 시 `SecureField`로 API 키 입력 행 노출.

### OCROverlayController (변경)
- `startTranslation()`: 설정이 `.deepl`이면 `DeepLTranslator(apiKey: settings.deepLAPIKey)`, 아니면 보유 중인 `appleTranslator`로 디스패치(둘 다 `Translator`). `.translationTask`는 애플 엔진에 계속 바인딩(DeepL 땐 idle).

## 데이터 흐름

`T` → 설정 엔진 선택 → `Translator.translate(원문)` → (DeepL이면 URLSession, 애플이면 세션 브리지) → 원문 아래 표시 / 실패 시 카드 오류.

## 검증

- `swift build` 성공.
- DeepL 방향·엔드포인트 판별 self-check(`assert`) 통과.
- 설정에서 DeepL 선택 + 키 입력 → 한/영 캡처 번역 확인. 키 지우면 오류 메시지 확인.
- 애플 엔진으로 되돌려도 기존대로 동작 확인.
