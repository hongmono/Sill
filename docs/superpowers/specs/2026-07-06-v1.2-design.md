# Sill v1.2 — 캡처 경험 (설계)

날짜: 2026-07-06 / 상태: 승인됨

## 범위

1. **썸네일 우클릭 메뉴** — AppKit `NSMenu` (DragNSView의 rightMouseDown):
   - "클립보드에 복사": 파일 URL + 이미지를 NSPasteboard에
   - "다른 이름으로 저장...": NSSavePanel로 파일 복사
2. **저장 위치·형식 설정** (일반 pane):
   - 파일 형식: PNG(기본)/JPEG — `screencapture -t`, 파일 확장자에 반영
   - 저장 위치: 기본 nil = 임시(App Support, 스택에서 빠지면 삭제). 폴더 지정 시 그 폴더에 영구 저장 — ✕/드래그는 스택에서만 제거하고 **파일은 유지**
3. **타이머 전체 캡처** — 메뉴 서브메뉴 "타이머 전체 캡처 ▸ 3초/10초" (`screencapture -T n`)

## 핵심 결정

- 파일 삭제 정책은 `Screenshot.isManaged`(임시 폴더 소속 여부)로 캡처 시점에 확정.
- 우클릭 메뉴는 SwiftUI `.contextMenu`가 아닌 AppKit — DragNSView가 이미 마우스 이벤트를 소유하므로 히트테스트 충돌 회피.
- 타이머는 전체 화면만 (영역 선택은 대화형이라 지연 무의미).

## 제외 (YAGNI)

임의 초 입력, 타이머 영역 캡처, JPEG 품질 설정, 파일명 패턴 설정.

## 구성

- 수정: `AppSettings.swift`(+imageFormat, saveDirectory), `CaptureService.swift`(형식/폴더/-T), `ScreenshotStore.swift`(isManaged 삭제 정책, copy/saveAs), `StackView.swift`(우클릭 메뉴 연결), `GeneralSettingsView.swift`(설정 UI), `AppDelegate.swift`(타이머 서브메뉴)
- 의존성 추가 없음. 자동 테스트 없음(기존 결정), 게이트 = `swift build` + 수동 검증.
