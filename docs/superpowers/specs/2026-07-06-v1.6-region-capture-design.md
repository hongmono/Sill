# Sill v1.6 — 캡처 프리뷰 흐름 설계

날짜: 2026-07-06
상태: 승인됨 (구현 완료)

## 목표

⇧⌘4 영역/창 캡처 후, 바로 스택/OCR로 보내지 않고 **큰 프리뷰**를 띄워 사용자가 결정하게 한다.
Enter=스택(날아가는 애니메이션), OCR키(기본 O)=텍스트 추출, ESC=버리기.

## 배경 / 결정

- v1.5의 "캡처 중 ⌘ 유지 = OCR"은 screencapture가 키보드를 가로채고 App Nap이 폴링을 조여 **flaky**했다.
- 자체 캡처 엔진(전체화면 오버레이 + ScreenCaptureKit)도 시도했으나 **느리고** 검증된 screencapture를 대체하는 리스크가 컸다(멀티모니터·레티나·창모드 재구현).
- 결론: **결정을 캡처 후(프리뷰)로 옮기면** 캡처 중 키 감지가 불필요해진다. → 애플 `screencapture -i`(빠름·검증됨) 그대로 쓰고, 결과를 프리뷰로. ⌘ 감지 문제 소멸.

## 흐름

1. ⇧⌘4 → `screencapture -i`(드래그=영역, 스페이스=창) → 임시 파일 → NSImage 로드 후 임시 파일 삭제
2. **프리뷰**(CapturePreviewController): 포커스 모니터 중앙에 크게(화면 ~70%, 작은 선택은 최대 2배 확대). 하단 힌트 바.
3. 키:
   - **Enter/Space** → 스택으로 날아가는 애니메이션(창 프레임 축소+이동+페이드) 후 saveToStack → 스택 등장 애니로 이어짐
   - **OCR키**(기본 O) → OCR 카드(스택엔 안 넣음). **물리 키(keyCode) 매칭** → 한글 등 다른 입력 소스에서도 같은 자리 키('ㅐ') 동작
   - **ESC** → 버리기(그냥 닫힘)

## 컴포넌트

- `CapturePreviewController` — 프리뷰 창(borderless key panel) + 로컬 keyDown 모니터 + fly 애니메이션. `onKeep`/`onOCR`/`stackTargetProvider` 콜백.
- `CaptureService.captureRegionForPreview(completion:)` — screencapture -i → 임시 파일 → NSImage. `saveToStack(_:)` — keep 시 설정 폴더·포맷으로 저장 후 store.add.
- `StackPanelController.nextThumbnailFrame()` — fly 애니메이션 목적지(다음 썸네일 위치).
- `AppSettings.ocrKey`(기본 "o") + GeneralSettingsView 필드.

## 제외 / 폐기

- 자체 캡처 엔진(오버레이·ScreenCaptureKit) — 시도 후 폐기(느림·리스크).
- OCR 후 원래 앱에 자동 붙여넣기 — 접근성 권한 필요해서 이번엔 보류.
- 전체화면(⇧⌘3)·타이머는 프리뷰 없이 스택 직행(기존 유지).

## 검증

- `swift build` 성공.
- 수동: ⇧⌘4→선택→프리뷰, Enter=스택(fly), O('ㅐ' 포함)=OCR, ESC=버리기, 창모드(스페이스), 설정에서 OCR키 변경. 첫 캡처 정상(로그로 확인 완료).
