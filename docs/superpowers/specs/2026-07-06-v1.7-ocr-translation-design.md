# Sill v1.7 — OCR 번역 설계

날짜: 2026-07-06
상태: 승인됨

## 목표

OCR 결과 카드에서 `T` 키를 누르면 텍스트 언어를 자동감지해 원문 아래에 번역을 표시.
- 한국어 감지 → 영어로 번역
- 그 외(영어 등) → 한국어로 번역

## 범위

포함:
- OCR 카드에 `T` = 번역 온디맨드 트리거
- 애플 Translation 프레임워크(온디바이스, 무료, API키 불필요)로 번역
- `NLLanguageRecognizer`로 언어 자동감지 → 방향 결정
- 원문 아래에 번역문 표시(교체 아님)
- 엔진을 갈아끼울 수 있는 `Translator` 프로토콜 seam

제외:
- 대상 언어 수동 선택 UI (자동 규칙으로 충분, YAGNI)
- 설정 화면 노출, 엔진 선택 UI (두 번째 엔진 생길 때)
- 번역 결과 편집/재번역 (YAGNI)

## 제약

- **최소 타깃 macOS 15로 상향** (14→15). 프로그래밍 방식 `TranslationSession`/`.translationTask`가 macOS 15+ 전용. `Package.swift`·`Info.plist` 수정.
- 외부 패키지 의존성 금지 — Translation·NaturalLanguage 모두 시스템 프레임워크.
- 자동 테스트 없음 — 게이트는 `swift build` 성공 + 수동 GUI 검증.
- 기존 OCR 카드 동작(Enter/⌘C/ESC) 불변.

## 컴포넌트

### Translator (신규, 어댑터 seam)
```swift
protocol Translator {
    func translate(_ text: String) async throws -> String
}
```
- 방향 규칙을 어댑터 내부에 캡슐화. 나중에 `DeepLTranslator` 등이 이 프로토콜만 구현하면 교체.

### AppleTranslator (신규, `Translator` 구현)
- `NLLanguageRecognizer.dominantLanguage(for:)`로 감지 → 한국어면 target=`en`, 아니면 target=`ko`.
  *(ponytail: 혼합 텍스트는 dominant 기준 휴리스틱. 부족하면 그때 정교화)*
- 애플 `TranslationSession`은 SwiftUI `.translationTask`로만 획득 가능(직접 생성 불가) → continuation 브리지:
  - `ObservableObject`, `@Published var configuration: TranslationSession.Configuration?`.
  - `translate(_:)`: 감지·target 계산 → continuation 저장 → `configuration` 세팅 → await.
  - 카드 뷰의 `.translationTask(configuration)`가 세션을 받아 `session.translate()` 실행 후 continuation resume, `configuration=nil`.
- 첫 사용 시 언어모델 다운로드 시스템 프롬프트는 프레임워크가 자동 처리. 실패는 throw → 카드에 표시.

### OCROverlayController (수정)
- 표시 상태를 담는 경량 `OCRCardModel`(`ObservableObject`) 도입: `original`, `translation?`, `isTranslating`, `error?`.
  컨트롤러(⌘C 복사)와 카드 뷰가 공유.
- 키모니터에 `T`(keyCode 17) 추가 → `Task { model.isTranslating=true; model.translation = try await translator.translate(model.original) }`, 실패 시 `model.error`.
- ⌘C/Enter 복사: 번역이 있으면 `원문 + "\n\n" + 번역`, 없으면 원문. *(ponytail: 카드에 보이는 것을 복사)*

### OCRResultView (수정)
- `@ObservedObject model`, 애플 엔진의 `configuration`에 `.translationTask` 부착(애플 특화 한 줄).
  *(ponytail: 뷰의 애플 바인딩만 엔진 특화, 나머지는 프로토콜 뒤)*
- 원문 스크롤 아래에 구분선 + 번역문(있을 때). `isTranslating`이면 진행 표시, `error`면 "번역 실패: …".
- 하단 힌트에 `T 번역` 추가.

## 데이터 흐름

`T` 키 → 컨트롤러가 `translator.translate(original)` 호출 → AppleTranslator가 감지·config 세팅 → 뷰의 `.translationTask`가 세션으로 번역 → 결과를 model에 기록 → 원문 아래 표시.

## 검증

- `swift build` 성공.
- 한글 스크린샷 → `T` → 영어 번역이 원문 아래 표시.
- 영문 스크린샷 → `T` → 한국어 번역.
- ⌘C가 원문+번역 복사, ESC 닫기 수동 확인.
- (첫 실행) 언어모델 다운로드 프롬프트 정상 동작 확인.
